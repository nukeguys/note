# 특징 추출(Feature Extraction)이란 무엇인가?

## 1. 특징 추출의 기본 정의

특징 추출이란,

> **원본 데이터에서  
> 모델이 판단에 사용할 “중요한 정보”를 선택하거나 변환하는 과정**이다.

모델은 현실 세계를 직접 이해하지 못한다.  
항상 사람이 준비한 **특징(feature)** 만을 입력으로 받아 학습한다.

## 2. 전통적인 머신러닝에서 특징 추출의 의미

전통적인 머신러닝에서  
“사람이 특징을 지정한다”는 말의 의미는 단순히 컬럼을 고른다는 뜻이 아니다.

이는 사람이 **미리 다음을 가정한다는 것**이다:

- 어떤 정보가 문제 해결에 중요할지
- 어떤 형태로 표현되어야 의미가 살아날지
- 어떤 정보는 무시해도 될지

👉 즉, **무엇이 중요한 정보인지에 대한 판단을 사람이 선제적으로 내린다.**

모델은 이 판단을 의심하지 않고,  
그 틀 안에서만 학습과 예측을 수행한다.

## 3. 예시 ① 집값 예측

### 문제

집값을 예측하라

### 사람이 특징을 지정하는 방식

```md
입력 특징:

- 평수
- 방 개수
- 위치
- 건축 연도
```

여기에는 이미 강한 가정이 들어 있다:

- 집값은 이 변수들로 설명될 수 있다
- 이 외의 정보(예: 이웃 분위기, 소음, 향후 개발 가능성 등)는 중요하지 않다

👉 모델은 이 가정의 범위 안에서만 똑똑해질 수 있다.

## 4. 예시 ② 이미지 분류 (고양이)

### 전통적인 ML 접근

사람이 먼저 말한다:

- 귀가 뾰족한가?
- 수염이 있는가?
- 색 분포는 어떤가?
- 가장자리(edge)가 얼마나 많은가?

이를 수치화하여 입력한다:

```text
특징 벡터 = [
  귀 뾰족함 정도,
  수염 검출 점수,
  엣지 밀도,
  색 히스토그램
]
```

👉 이 특징들은
“이런 정보가 고양이를 판단하는 데 중요하다”는 사람의 가정 결과물이다.

## 5. 사람이 특징을 지정한다는 말의 핵심

정리하면 다음과 같다:

- 사람이 “중요함”의 기준을 미리 정의한다
- 모델은 그 기준을 바꾸거나 확장하지 못한다
- 잘못된 특징 선택은 성능 한계로 직결된다

이것이 전통적인 머신러닝의 구조적 제약이다.

## 6. 왜 이것이 전통 ML의 근본적 한계인가?

현실 세계에서는:

- 무엇이 중요한지 미리 알기 어렵고
- 문제마다 중요한 정보가 다르며
- 상황에 따라 중요도가 계속 변한다

하지만 전통 ML은:

- 사람이 정한 관점에서만 세상을 본다
- 새로운 관점을 스스로 만들어내지 못한다

👉 이 지점에서 확장성의 한계가 발생한다.

## 7. 딥러닝의 전환점: 특징 추출의 자동화

딥러닝은 질문을 이렇게 바꾼다:

❌ “어떤 특징이 중요할까?”
✅ “중요한 특징을 데이터로부터 학습하게 하자”

딥러닝 모델 내부에서는:

- 초기 층: 단순한 패턴 (선, 점, 색)
- 중간 층: 부분 구조 (귀, 눈, 질감)
- 깊은 층: 개념적 패턴 (고양이의 얼굴, 형태)

👉 무엇이 중요한지는 사람이 아니라 모델이 결정한다.

이것이 ML → DL로 넘어간 가장 큰 사고 전환이다.

## 8. 중요한 오해 하나 정리

사람이 특징을 직접 지정하지 않는다고 해서 사람의 역할이 사라진 것은 아니다.

사람은 여전히:

- 문제 정의
- 데이터 수집 방식
- 모델 구조 선택
- 학습 목표(손실 함수) 설정

을 통해 간접적으로 무엇이 중요할지를 유도한다.

👉 딥러닝은 “가정이 없는 학습”이 아니라  
👉 **“가정을 더 뒤로 미룬 학습”** 이다.

## 9. Week 1 관점에서의 핵심 요약

- 특징 추출 = 무엇을 중요한 정보로 볼지 결정하는 과정
- 전통 ML: 사람이 그 기준을 미리 정한다
- 딥러닝: 그 기준을 데이터에서 학습한다

이것이 규칙 기반 → 머신러닝 → 딥러닝으로 이어지는 핵심 변화다
