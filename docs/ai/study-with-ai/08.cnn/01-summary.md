# 1. CNN (Convolutional Neural Network)

## 1. 이 주차의 핵심 질문

이미지는 숫자일 뿐인데, 어떻게 의미(고양이, 사람)를 얻는가?

## 2. 문제의 본질 (출발점)

• 이미지 = 픽셀 값의 집합 (숫자 배열)
• 우리가 원하는 것 = 추상적 개념

👉 즉,

저수준 데이터 → 고수준 의미 변환 문제

## 3. 기존 방식(MLP)의 실패

❌ 문제 1: 공간 정보 붕괴

- 위치 관계가 사라짐

❌ 문제 2: 파라미터 폭발

- 입력 크기 커지면 학습 불가능

👉 결론:

- 이미지는 일반 신경망으로 다루기 부적합

## 4. CNN의 핵심 아이디어 (이거 하나)

“전체를 보지 말고, 부분(local)부터 이해하자”

## 5. Convolution의 본질

정의 (직관)

- 필터 = 특정 패턴을 감지하는 센서

동작

- 작은 영역을 본다
- 필터를 적용한다
- 전체를 훑는다

결과:

- 이미지 → feature map (패턴의 강도 지도)

## 6. CNN의 2가지 핵심 원리

1. Locality (지역성)

- 가까운 픽셀끼리만 본다

2. Weight Sharing (가중치 공유)

- 같은 필터를 전체에 사용

👉 효과:

- 파라미터 감소
- 일반화 증가
- 위치 이동에도 강함

## 7. CNN의 진짜 핵심: 계층적 표현 (⭐)

CNN은 한 번에 이해하지 않는다.

표현의 변화 과정

pixel  
→ edge  
→ texture  
→ part (눈, 코)  
→ object (얼굴)

👉 핵심:

의미는 쌓인다 (hierarchical)

## 8. 가장 중요한 관점 전환 (진짜 핵심)

CNN은 “이미지를 이해”하지 않는다  
👉 표현을 점점 더 의미 있게 바꾼다

## 9. Pooling의 역할

위치에 덜 민감하게 만든다

예:

- 왼쪽 고양이 vs 오른쪽 고양이 → 동일 처리

## 10. Transfer Learning (실무 핵심)

왜 가능한가?

- 초기 레이어:
- edge, texture → 모든 이미지 공통

👉 구조:

- 앞부분 = 범용
- 뒷부분 = 문제 특화

👉 결과:

- 이미 학습된 모델을 재사용 가능

## 11. CNN이 남긴 중요한 변화

이게 진짜 중요합니다.

Before (MLP 시대)

- 하나의 모델로 모든 문제 해결 시도

After (CNN 이후)

- 문제 구조에 맞는 모델 설계

👉 흐름:

- 이미지 → CNN
- 순서 → RNN
- 전체 관계 → Transformer

## 12. CNN의 한계

❌ 전역 관계 이해 약함

❌ 위치 변화(회전 등)에 취약

❌ receptive field 제한

👉 그래서 등장:

Transformer

## 13. 한 문장 핵심 정리

CNN은 이미지에서 의미를 찾는 것이 아니라
지역 패턴을 계층적으로 조합하여 의미를 만든다

### 반드시 기억할 관점

CNN은 의미를 이해하는 것이 아니라,
표현을 점진적으로 변환하는 구조다.
