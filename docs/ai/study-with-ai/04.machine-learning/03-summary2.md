# 03. Week 3–4 Summary

## 1. 모델(Model)이란 무엇인가

### 모델의 정체

- **모델은 함수이다**
- 수식으로 표현하면:

```text
y = f(x; θ)
```

- `f` : 모델 구조 (가설 공간, architecture)
- `θ` : 학습을 통해 결정되는 파라미터
- `y` : 예측 결과 (모델이 아님)

### 모델은 두 단계로 존재한다

1. **학습 전 모델**
   - 구조만 정의된 상태
   - 아직 데이터를 통해 학습되지 않음
   - Bias–Variance의 *경향*을 내포

2. **학습된 모델**
   - 데이터로부터 파라미터가 결정된 상태
   - 실제 예측을 수행
   - Bias–Variance가 *행동 특성*으로 드러남

## 2. 학습(Training)이란 무엇인가

- 학습은 단일 입력에 대한 함수 적용이 아니다
- 정확한 표현:

```text
Train(D) → Model (θ̂)
```

- `D` : 학습 데이터셋
- `θ̂` : 학습된 파라미터

### 학습의 본질

- **손실 함수(Loss Function)를 최소화하는 과정**
- 즉, 오차를 줄이는 방향으로 파라미터를 반복적으로 조정

## 3. 성능(Performance)이란 무엇인가

### 성능은 결과로만 판단한다

- 모델 내부가 아니라 **출력 결과**만을 관측
- 핵심 지표는 두 가지

#### ① 평균 성능 (Mean)

- 평균 정확도, 평균 손실
- 전반적인 예측 수준

#### ② 결과 분산 (Statistical Variance of Results)

- 평가를 반복했을 때 결과의 흔들림
- 성능의 **안정성**

👉 **성능이 좋다 = 평균이 높고, 분산이 작다**

## 4. Bias와 Variance란 무엇인가

### 기본 관점

- **Bias와 Variance는 직접 측정하는 값이 아니다**
- 관측된 결과를 설명하기 위한 **해석 프레임**이다

### Bias (편향)

- 의미:
  - 모델이 현실을 지나치게 단순화함
  - 항상 **비슷한 방식으로 틀리는 경향**

- 관측 신호:
  - Train / Test 성능 모두 낮음
  - 특정 유형의 문제를 반복적으로 틀림

- 해석:
  - 구조적 한계
  - **과소적합(Underfitting)**

### Variance (모델 분산)

- 의미:
  - 데이터 변화에 대한 과민 반응
  - 모델 행동의 **불안정성**

- 관측 신호:
  - Train 성능은 높음
  - Test 성능은 낮고 들쭉날쭉함

- 해석:
  - **과적합(Overfitting)**

## 5. Bias–Variance와 평균·분산의 관계

### 개념 구분

- **Bias / Variance** → 원인 (모델의 성향)
- **평균 / 결과 분산** → 관측 결과

### 인과 구조

```text
모델 구조 + 학습 과정 + 데이터
            ↓
       Bias / Variance
            ↓
   평균 성능 / 결과 분산
```

- Variance가 크면 → 결과 분산이 커지는 경향
- Bias가 크면 → 평균 성능이 낮아지는 경향

👉 우리는 **결과를 보고 Bias·Variance를 추정**한다

## 6. 시험 비유로 정리

- 시험 점수가 전반적으로 낮다
  → 평균 낮음
  → Bias 의심

- 시험 점수가 들쭉날쭉하다
  → 결과 분산 큼
  → Variance 의심

- 비슷한 문제를 계속 틀린다
  → Bias가 큰 성향

- 시험마다 점수 차이가 크다
  → Variance가 큰 성향

## 7. 핵심 문장 요약

> **Bias와 Variance는
> 결과(평균·분산)를 설명하기 위한
> 모델의 성향에 대한 해석 프레임이다.**
