# 1. Summary — 딥러닝 → CNN → Transformer → LLM

## 1️⃣ 출발점: 딥러닝은 왜 성공했는가

### 핵심

딥러닝은 새로운 기술이 아니라
실행 가능해진 기술

### 이유 (3가지)

- 데이터 → 학습 재료
- GPU → 학습 속도 (실험 가능)
- 알고리즘 → 학습 안정성

### 🔥 핵심 관점

성능 향상이 아니라
“학습 자체가 가능해진 것”

## 2️⃣ 문제: 데이터는 구조를 가진다

### 중요한 전환

데이터는 그냥 숫자가 아니다

### 예

- 이미지 → 공간 구조
- 문장 → 순서 + 관계

👉 그래서:

모든 문제를 같은 모델로 풀 수 없음

## 3️⃣ CNN — 구조를 이용하는 모델

### 핵심

이미지는
local 패턴의 조합

### 동작

- 작은 필터 → 패턴 탐지
- 필터 공유 → 위치 상관없이 탐지
- 여러 필터 → 다양한 특징 추출
- 계층 → 작은 패턴 → 큰 의미

### 🔥 본질

CNN은
구조를 “가정”하고 효율을 얻는다

### 장점

- 빠름
- 데이터 적어도 됨
- 안정적

### 한계

멀리 떨어진 관계를 직접 못 봄

## 4️⃣ Attention — 중요한 것 선택

### 핵심

모든 정보를 똑같이 보지 않는다
→ 중요도 계산

### 본질

상황(context)에 따라
중요도가 바뀐다

### 🔥 핵심

Attention = “지금 무엇이 중요한가?”

## 5️⃣ Transformer — 구조의 변화

### 핵심

전체를 한 번에 보고
모든 관계를 직접 계산

### 특징

- 모든 요소 ↔ 모든 요소 연결
- 병렬 처리 가능
- 장거리 관계 해결

### 🔥 본질

Transformer는
구조를 가정하지 않고 관계를 학습한다

## 6️⃣ CNN vs Transformer (중요 비교)

| 항목   | CNN         | Transformer     |
| ------ | ----------- | --------------- |
| 접근   | 구조 가정   | 구조 학습       |
| 범위   | local 중심  | global 직접     |
| 데이터 | 적어도 가능 | 많이 필요       |
| 계산량 | 적음        | 큼              |
| 속도   | 빠름        | 상대적으로 느림 |

### 🔥 핵심 한 줄

CNN = 효율  
Transformer = 표현력

## 7️⃣ LLM의 본질

### 정의

LLM은 다음 토큰을 확률적으로 예측하는 모델

### 내부 동작

```text
문맥
↓
Attention (중요도 계산)
↓
확률 분포 생성
↓
다음 토큰 선택
```

### 🔥 핵심 관점

이해하는 게 아니라 관계를 기반으로 예측한다

## 8️⃣ 가장 중요한 통찰 (Week 7 핵심)

### 🔥 전체 흐름

```text
딥러닝 가능
↓
CNN (구조 활용)
↓
한계 (global 부족)
↓
Attention (중요도)
↓
Transformer (관계 모델)
↓
LLM (확률적 생성)
```

### 🎯 최종 핵심 문장

현대 AI는 데이터를 이해하는 시스템이 아니라 문맥에서 중요한 관계를 계산해 다음 출력을 예측하는 시스템이다.
