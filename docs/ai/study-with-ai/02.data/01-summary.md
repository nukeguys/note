# 02. Summary

```md
## 핵심 전제

- 세상은 명시적인 규칙으로 완전히 설명할 수 없다.

## 반드시 기억해야 할 관점

- 머신러닝은 규칙을 정의하지 않고,
  새로운 상황에서 덜 틀리는 행동을 학습한다.

## 대표 질문

- 지능은 내부 구조로 판단해야 하는가,
  아니면 보지 못한 상황에 대한 대응으로 판단해야 하는가?
```

```text
[현실 세계]
 └─ 복잡성 / 불확실성 / 예외의 폭발
        ↓
[규칙 기반 접근]
 ├─ 명시적 규칙 필요
 ├─ 규칙 추가 = 복잡도 증가
 └─ 유지·확장 불가능
        ↓ (한계)
────────────────────────
        ↓
[발상의 전환]
 ├─ 규칙을 쓰지 말자
 ├─ 사람이 설명하지 말자
 └─ 결과(정답)만 주자
        ↓
[머신러닝]
 ├─ 데이터 (과거 경험)
 ├─ 학습 (경향성 추출)
 └─ 모델 (행동 함수)
        ↓
────────────────────────
[좋은 모델이란?]
 ├─ 과거 데이터에 잘 맞는가?
 │    └─ Training Performance
 ├─ 새로운 데이터에서도 잘 작동하는가?
 │    └─ Generalization
        ↓
[문제 발생]
 ├─ 과거에만 너무 잘 맞음
 │    └─ Overfitting
 ├─ 새로운 데이터에 약함
 │    └─ Generalization Failure
        ↓
────────────────────────
[필연적 질문들]
 ├─ 왜 데이터를 나눠야 하지?
 │    └─ Train / Test (Validation)
 ├─ “잘 맞는다”를 어떻게 판단하지?
 │    └─ Evaluation Metric / Loss
 ├─ 어느 정도가 적당한가?
 │    └─ Bias vs Variance

```
