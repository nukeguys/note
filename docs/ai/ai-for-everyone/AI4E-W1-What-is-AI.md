## 3. 섹션별 핵심 내용 정리

### Section 1: What is AI?

#### AI의 경제적 가치

McKinsey Global Institute 연구에 따르면, AI는 2033년까지 매년 **13조~22조 달러**의 추가 가치를 창출할 것으로 예상된다.

**가치 창출 구조:**

- **Generative AI (생성형 AI)**: 약 3~4조 달러
- **기타 AI 형태** (주로 Supervised Learning): 나머지 대부분

생성형 AI가 많은 주목을 받고 있지만, 실제로는 지도 학습과 같은 다른 AI 형태가 더 큰 경제적 가치를 창출한다.

**산업 전반 영향:** AI는 소프트웨어 산업뿐만 아니라 거의 모든 산업을 변화시킨다:

- 소매 (Retail)
- 여행 (Travel)
- 운송 (Transportation)
- 자동차 (Automotive)
- 제조 (Manufacturing)

**비전형적 산업 예시 - 미용업:** 로봇이 사람의 머리를 자르는 것은 기술적으로 매우 어렵지만, AI는 여전히 이 산업에 영향을 미칠 수 있다:

- 고객에게 어울리는 헤어스타일 추천
- 비즈니스 운영 최적화
- 예약 시스템 개선

#### AI의 구분

AI에 대해 이야기할 때 실제로는 두 가지 매우 다른 개념이 존재한다.

**ANI (Artificial Narrow Intelligence / 인공 좁은 지능)**

**정의:** 한 가지 특정한 작업만 수행하도록 설계된 AI

**예시:**

- 스마트 스피커 (Smart Speaker): 음성 명령 인식 및 응답
- 자율 주행 자동차 (Self-driving Car): 운전
- 웹 검색 (Web Search): 정보 검색
- AI 기반 농업 (AI in Farming): 작물 관리

**경제적 중요성:** 현재 AI가 창출하는 거의 모든 경제적 가치는 ANI에서 발생한다. ANI가 현재 실제로 구축하고 배포할 수 있는 기술이다.

**AGI (Artificial General Intelligence / 인공 일반 지능)**

**정의:** 인간이 할 수 있는 모든 지적 작업을 수행할 수 있거나, 그 이상을 해낼 수 있는 AI

**현실성:**

- AGI는 아직 존재하지 않는다
- 실현까지 수십 년, 어쩌면 수백 년이 걸릴 수 있는 먼 미래의 목표다
- 많은 연구자들이 AGI를 연구하고 있지만, 언제 실현될지는 아무도 모른다

이 강의는 주로 ANI에 집중한다. ANI가 오늘날 실제로 가치를 창출하고 있기 때문이다.

#### Generative AI (생성형 AI)

**정의:** 텍스트, 이미지, 오디오 등 고품질 콘텐츠를 생성할 수 있는 AI 기술

**대표 예시:**

- **ChatGPT**: 텍스트 생성
- **이미지 생성 도구**: DALL-E, Midjourney 등
- **오디오 생성**: 음성 합성, 음악 생성

**특징:** 생성형 AI는 ANI의 일종이지만 특별한 점이 있다. 한 가지 작업(예: 텍스트 생성)만 수행하지만 매우 범용적으로 사용될 수 있다:

- 텍스트 요약
- 이메일 작성
- 브레인스토밍 파트너

**분류:** 생성형 AI는 ANI와 AGI 사이 어딘가에 위치한다고 볼 수 있지만, 기술적으로는 여전히 ANI다.

### Section 2: Machine Learning

#### Supervised Learning (지도 학습)

**정의:** 입력(Input A)에서 출력(Output B)으로의 매핑(mapping)을 학습하는 머신러닝 방법

**중요성:** 지도 학습은 현재 가장 널리 사용되고 가장 많은 경제적 가치를 창출하는 머신러닝 유형이다.

#### A to B Mapping 예시

**1. 스팸 필터 (Spam Filtering)**

- Input A: 이메일 (Email)
- Output B: 스팸 여부 (0 또는 1)
- 원리: 이메일을 입력하면 AI가 스팸인지 아닌지 출력한다

**2. 음성 인식 (Speech Recognition)**

- Input A: 오디오 클립 (Audio clip)
- Output B: 텍스트 스크립트 (Text transcript)
- 원리: 오디오를 입력하면 시스템이 텍스트를 출력한다

**3. 기계 번역 (Machine Translation)**

- Input A: 영어 (English)
- Output B: 중국어 (Chinese)
- 원리: 영어 문장을 입력하면 중국어 번역을 출력한다

**4. 온라인 광고 (Online Advertising)**

- Input A: 광고 정보 + 사용자 정보 (Ad info + User info)
- Output B: 클릭 여부 (0 또는 1)
- 특징: 매우 수익성이 높은 응용 분야로, 구글, 페이스북 등 대형 기술 회사 수익 모델의 핵심이다

**5. 자율 주행 (Self-driving Cars)**

- Input A: 차량 전방 이미지 + 레이더 정보
- Output B: 다른 차량의 위치
- 원리: 자동차의 센서 데이터를 입력하면 주변 차량의 위치를 출력한다

**6. 시각적 검사 - 제조업 (Visual Inspection - Manufacturing)**

- Input A: 휴대폰 사진
- Output B: 결함 여부 (0 또는 1)
- 원리: 제조 공정에서 제품 사진을 찍으면 AI가 스크래치나 결함 유무를 판단한다

**핵심 원리:** 이 모든 예시의 공통점은 입력 A를 주면 출력 B를 얻는다는 것이다. 이것이 지도 학습의 핵심이다.

#### Large Language Model (LLM / 대규모 언어 모델)

**정의:** 방대한 양의 텍스트 데이터로 훈련되어, 주어진 단어들 다음에 올 단어를 예측하는 AI 모델

**ChatGPT의 작동 원리:**

**학습 방식:**

1. 인터넷에서 수집한 방대한 텍스트 데이터 사용
2. 문장의 일부를 입력(A)으로 제공
3. 시스템이 다음 단어를 출력(B)으로 예측하도록 학습
4. 이 과정을 반복

**구체적 예시:** 문장: "My favorite drink is lychee bubble tea"

- "My favorite drink is"를 입력(A)으로 제공 → "lychee"를 정답(B)으로 예측하도록 학습
- "My favorite drink is lychee"를 입력 → "bubble"을 예측
- 이런 식으로 반복

**핵심 통찰:** ChatGPT는 기본적으로 지도 학습을 사용한다. 입력(앞의 단어들)에서 출력(다음 단어)으로의 매핑을 학습한 것이다. 이것을 수백억, 수천억 번 반복하면서 엄청난 양의 지식을 습득하게 된다.

**결과:** 이 간단한 원리를 대규모로 적용하면, 시스템이 언어 패턴, 사실, 추론 능력까지 학습하게 된다. 이것이 생성형 AI의 기초다.

### Section 3: Why Now?

#### 데이터와 성능의 관계

**과거의 한계 - 전통적 AI 알고리즘:**

과거의 전통적인 머신러닝 알고리즘들은 중요한 한계가 있었다.

**성능 곡선 특징:**

- X축: 데이터 양 (Amount of data)
- Y축: 성능 (Performance)
- 전통적 AI의 곡선: 초기에는 성능이 향상되지만, 일정 시점 이후 **평탄화(plateau)** 된다

**의미:** 데이터를 아무리 많이 수집해도, 특정 시점 이후에는 성능이 더 이상 좋아지지 않았다. 이것이 과거 AI의 큰 제약이었다.

#### Neural Networks (신경망)의 혁신

신경망, 특히 딥러닝이 이 상황을 완전히 바꿔놓았다.

**신경망 크기별 특징:**

**작은 신경망 (Small Neural Network):**

- 데이터가 늘면 성능이 향상됨
- 하지만 여전히 어느 정도 한계가 있음

**중간 신경망 (Medium Neural Network):**

- 더 많은 데이터에서 더 좋은 성능
- 작은 신경망보다 높은 성능 상한선

**큰 신경망 (Large Neural Network):**

- **핵심 특징**: 데이터가 매우 많을수록 성능이 계속 향상됨
- 평탄화 현상이 거의 없음
- 오늘날의 딥러닝 시스템이 이 범주에 속함

**그래프 해석:** 큰 신경망은 데이터가 많으면 많을수록 계속해서 성능이 좋아진다. 이것이 현대 AI의 핵심 원리다.

#### AI 부상의 두 가지 핵심 요건

**1. Big Data (빅데이터)**

**배경:** 디지털 사회가 되면서 엄청난 양의 데이터를 생성하고 있다:

- 인터넷 활동
- 모바일 기기
- IoT 센서
- 디지털화된 비즈니스 프로세스

**중요성:** 이 데이터가 없었다면, 큰 신경망을 훈련시킬 수 없었을 것이다.

**2. Computational Power (컴퓨팅 파워)**

큰 신경망을 훈련시키려면 엄청난 계산 능력이 필요하다.

**핵심 기술:**

- **GPU (Graphics Processing Unit)**: 원래 비디오 게임 그래픽을 위해 개발되었지만, 신경망 훈련에 매우 효과적임이 밝혀졌다
- 클라우드 컴퓨팅
- 전문 AI 칩

**역사적 맥락:** 10년 전에는 큰 신경망을 훈련시키는 것이 기술적으로나 경제적으로 불가능했다. 하지만 GPU와 클라우드 컴퓨팅의 발전으로 이것이 가능해졌다.

**결론:**

AI가 지금 부상하는 세 가지 이유:

1. 충분한 데이터를 갖게 되었다
2. 큰 신경망을 훈련시킬 수 있는 컴퓨팅 파워를 갖게 되었다
3. 신경망이 더 많은 데이터로 계속 학습할 수 있다는 것을 발견했다

이 세 가지가 결합되어 오늘날의 AI 혁명을 만들어냈다.

### Section 4: What is Data?

#### 데이터의 정의

**기본 개념:** 데이터는 AI 시스템을 구축하는 핵심 재료(raw material)다.

**구조:** 어떤 AI 시스템을 만들고 싶다면, 먼저 입력(A)과 출력(B)을 정의해야 한다. 그리고 나서 이 A와 B의 예시들을 모은 것이 데이터 세트(dataset)다.

#### 데이터의 유형

**1. Structured Data (정형 데이터)**

**정의:** 데이터베이스나 엑셀 스프레드시트처럼 표(table) 형태로 정리된 데이터

**주택 가격 예측 예시:**

| Size (sq ft) | # Bedrooms | Price ($) |
| ------------ | ---------- | --------- |
| 523          | 1          | 100,000   |
| 1,503        | 3          | 230,000   |
| 2,100        | 4          | 315,000   |

**구조:**

- 각 행(row): 하나의 데이터 포인트 (한 채의 집)
- 각 열(column): 하나의 속성 (크기, 침실 수, 가격)

**특징:** 정형 데이터는 오래전부터 사용되어 왔다. 전통적인 데이터베이스와 분석에서 주로 다루던 형태다.

**2. Unstructured Data (비정형 데이터)**

**정의:** 이미지, 오디오, 텍스트처럼 정해진 표 형태가 없는 데이터

**예시:**

- **이미지**: 사진, 의료 영상(X-ray, MRI)
- **오디오**: 음성 녹음, 음악
- **텍스트**: 이메일, 문서, 소셜 미디어 포스트
- **비디오**: 영상 클립

**중요성:** 최근의 AI 붐은 주로 비정형 데이터를 처리할 수 있게 되면서 일어났다. 과거에는 이런 형태의 데이터를 컴퓨터가 이해하기 매우 어려웠다.

**인간 vs 컴퓨터:** 인간은 이미지나 오디오를 자연스럽게 인식한다. 하지만 컴퓨터에게는 매우 어려운 작업이었다. 딥러닝이 이 문제를 해결했다.

#### 데이터 획득 방법

**1. Manual Labeling (수동 라벨링)**

**정의:** 사람이 직접 데이터를 보고 태그(정답)를 다는 방식

**고양이 vs 개 분류 예시:**

- 1,000장의 동물 사진을 수집한다
- 사람이 각 사진을 보고 '고양이' 또는 '개'라고 라벨을 단다
- 이렇게 라벨이 달린 데이터로 AI를 훈련시킨다

**특징:** 수동 라벨링은 시간과 비용이 많이 든다. 하지만 많은 경우 필수적이다. 특히 초기 단계에서 그렇다.

**2. Observing Behaviors (행동 관찰)**

**정의:** 사용자의 행동이나 기계의 작동을 관찰하여 자동으로 데이터를 수집하는 방식

**웹사이트 사용자 행동 예시:** 이커머스 사이트를 운영한다면:

- Input A: 사용자에게 보여준 제품 정보
- Output B: 사용자가 구매했는지 여부 (0 또는 1)
- 이 데이터는 자동으로 수집된다. 별도의 수동 라벨링이 필요 없다

**기계 센서 데이터 예시:** 제조 공장의 기계에서:

- Input A: 기계의 센서 측정값 (온도, 진동, 압력 등)
- Output B: 기계가 고장났는지 여부
- 센서가 자동으로 데이터를 기록한다

**장점:** 행동 관찰 방식은 수동 라벨링보다 확장성이 훨씬 좋다. 시스템이 운영되면서 자동으로 데이터가 쌓이기 때문이다.

**3. Download / Partnership (다운로드 / 파트너십)**

**공개 데이터셋 다운로드:** 많은 경우, 누군가가 이미 만들어 놓은 데이터셋을 사용할 수 있다:

- 학술 연구용 공개 데이터
- 정부 공개 데이터
- 오픈 소스 데이터셋

**파트너십 예시:** 의료 AI를 개발하는 회사라면, 병원과 파트너십을 맺어 의료 영상 데이터를 제공받을 수 있다. 물론 환자 개인정보 보호를 준수해야 한다.

#### 데이터의 오용 (Misuse of Data)

**잘못된 접근 1 - IT 인프라 우선 투자**

**문제 상황:** '우리가 데이터를 수집할 수 있는 모든 IT 시스템을 먼저 구축하자. 그러면 나중에 AI 팀이 와서 가치 있는 것을 발견할 것이다.'

**실패 이유:** 이것은 거의 항상 실패한다:

1. 어떤 데이터가 가치 있는지 모르고 무작정 수집하게 된다
2. 나중에 실제로 필요한 데이터가 없을 수 있다
3. 막대한 IT 투자가 낭비된다

**올바른 접근:** 초기부터 AI 팀과 협력해야 한다. AI 팀이 '우리에게는 이런 데이터가 필요합니다'라고 말하면, 그것을 수집하는 시스템을 구축한다. 피드백 루프가 중요하다.

**잘못된 접근 2 - 데이터 쌓아두기**

**문제 상황:** '데이터는 새로운 석유다'라는 말을 듣고, 무조건 데이터를 많이 모으기만 하면 가치가 생길 것이라고 생각한다.

**현실 - Garbage in, garbage out:** 나쁜 품질의 데이터를 아무리 많이 모아도 가치가 없다. 오히려 문제만 발생한다.

#### 데이터 품질 문제 (Messy Data)

**문제 1 - 잘못된 라벨 (Incorrect Labels)**

주택 가격 데이터 예시:

| Size | Bedrooms | Price   | Label Quality   |
| ---- | -------- | ------- | --------------- |
| 523  | 1        | 100,000 | ✓ Correct       |
| 1503 | 3        | ???     | ✗ Wrong/Missing |

데이터 수집 과정에서 실수가 발생할 수 있다. 사람이 라벨을 잘못 달거나, 자동 수집 시스템에 오류가 있을 수 있다.

**문제 2 - 누락된 값 (Missing Values)**

데이터셋에 빈 칸이 있을 수 있다:

- 어떤 집은 침실 수가 기록되지 않았을 수 있다
- 센서가 고장나서 측정값이 없을 수 있다

**처리 방법:** AI 팀은 이런 데이터를 정제(clean)해야 한다:

- 잘못된 데이터 제거
- 누락된 값 보완 (평균값 사용, 예측 등)
- 이상치(outlier) 제거

**핵심 메시지:** 데이터의 양도 중요하지만, 품질이 더 중요하다. 깨끗하고 정확한 1,000개의 데이터가 지저분한 10,000개의 데이터보다 나을 수 있다.

### Section 5: The Terminology of AI

#### Machine Learning (머신러닝)

**정의:** 컴퓨터가 명시적으로 프로그래밍되지 않고도 학습할 수 있는 능력을 주는 과학 분야

**전통적 프로그래밍 vs 머신러닝:**

**전통적 프로그래밍:**

- 프로그래머가 모든 규칙을 코드로 작성한다
- 예: '만약 이메일에 "무료"라는 단어가 5번 이상 나오면 스팸으로 분류하라'

**머신러닝:**

- 데이터(예시)를 주면, 컴퓨터가 스스로 규칙을 학습한다
- 더 정확하고 복잡한 패턴을 찾을 수 있다

**실용적 관점:** 머신러닝은 입력 A를 받아 출력 B를 내놓는 소프트웨어를 만드는 것이다. 앞서 본 모든 A→B 매핑 예시들이 머신러닝이다.

**응용 분야:**

- 광고 추천 시스템
- 자율 주행
- 의료 진단
- 제조업 품질 관리

#### Data Science (데이터 과학)

**정의:** 데이터에서 지식과 통찰력(insights)을 추출하는 과학 분야

**머신러닝과의 차이:**

**부동산 개발 회사 시나리오:**

**머신러닝 접근:** 어떤 집을 지어야 수익이 날지 자동으로 예측하는 소프트웨어를 만든다:

- Input: 위치, 크기, 설계 등
- Output: 예상 수익
- 결과물: 실행되는 소프트웨어/앱

**데이터 과학 접근:** 과거 데이터를 분석하여 경영진의 의사결정을 돕는 보고서를 만든다:

- 분석: 어떤 지역이 가장 수익성이 높은가?
- 분석: 집을 리모델링하면 가격이 얼마나 오르는가?
- 결과물: 슬라이드(PowerPoint), 보고서

**여행사 예시:**

**머신러닝:** 고객 데이터를 보고 자동으로 여행 상품을 추천하는 시스템

**데이터 과학:** "우리 고객의 70%가 해변 휴양지를 선호한다"는 인사이트를 찾아 경영진에게 보고하고, "따라서 해변 관련 마케팅에 더 투자해야 한다"는 전략을 제안

**핵심 차이점:** 머신러닝은 자동화된 시스템을 만드는 것이고, 데이터 과학은 인간의 의사결정을 돕는 인사이트를 만드는 것이다. 물론 두 분야는 많이 겹치고, 같은 도구를 사용하기도 한다.

#### Deep Learning (딥러닝) / Neural Network (신경망)

**용어 관계:** 딥러닝과 신경망은 거의 같은 의미로 사용된다. 기술적으로는 딥러닝이 깊은(여러 층의) 신경망을 의미하지만, 실무에서는 두 용어를 교환해서 사용한다.

**역사적 배경:** 신경망이라는 이름은 인간의 뇌(뉴런)에서 영감을 받았다.

**오해 바로잡기:** 많은 사람들이 신경망이 뇌처럼 작동한다고 생각하지만, 이것은 거의 맞지 않다:

- 신경망은 실제로는 복잡한 수학 방정식의 집합이다
- 생물학적 뇌와는 거의 관련이 없다
- '뇌에서 영감을 받았다'는 것은 마케팅에 가깝다

**실용적 이해:** 신경망은 머신러닝과 데이터 과학을 수행하는 매우 강력한 도구다. 특히 이미지, 오디오, 텍스트 같은 비정형 데이터를 다룰 때 탁월하다.

**위치:** 딥러닝은 머신러닝의 한 종류다. 그리고 데이터 과학에서도 딥러닝을 사용할 수 있다.

#### 기타 AI 용어들 (AI Buzzwords)

[슬라이드 항목 – 간단히 언급만 됨]

다음 용어들이 나열되었으나 구체적 정의는 설명되지 않음:

- Unsupervised Learning (비지도 학습)
- Reinforcement Learning (강화 학습)
- Graphical Models (그래픽 모델)
- Planning (계획)
- Knowledge Graph (지식 그래프)

**언급 내용:** 이 모든 용어들을 알 필요는 없다. 이것들은 AI를 수행하는 다양한 도구들일 뿐이다. 누군가가 이런 용어들을 사용하며 복잡하게 말한다면, '아, 그것도 AI를 하는 또 다른 방법이구나'라고 생각하면 된다.

**핵심:** 중요한 것은 세부 기술 용어가 아니라, AI가 무엇을 할 수 있고, 비즈니스에 어떻게 적용할 수 있는지를 이해하는 것이다.

### Section 6: What makes an AI company?

#### 인터넷 시대의 교훈

**잘못된 공식:** 쇼핑몰 + 웹사이트 = 인터넷 기업? 아니다.

**배경:** 1990년대, 2000년대 초반에 많은 전통적인 기업들은 '우리도 웹사이트를 만들면 인터넷 기업이 될 수 있다'고 생각했다. 하지만 단순히 웹사이트를 만드는 것만으로는 진정한 인터넷 기업이 될 수 없었다.

**진정한 인터넷 기업의 특징:**

인터넷이 가능하게 한 새로운 방식들:

**1. A/B Testing (AB 테스트)** 웹사이트에서 두 가지 버전을 동시에 테스트하여 어떤 것이 더 효과적인지 빠르게 파악할 수 있다. 전통적인 매장에서는 불가능했던 일이다.

**2. Short Iteration Times (짧은 반복 주기)** 소프트웨어를 매일, 매주 업데이트할 수 있다. 전통적인 제품 개발 주기(몇 개월, 몇 년)와는 완전히 다르다.

**3. Decision-making pushed down (의사결정 권한 위임)** 엔지니어와 제품 매니저가 빠르게 실험하고 결정을 내릴 수 있다. 모든 결정을 최고 경영진이 내리지 않는다.

**핵심:** 진정한 인터넷 기업은 인터넷이 잘하는 일들을 수행하도록 조직된 회사다. 단순히 인터넷 기술을 사용하는 것이 아니다.

#### AI 기업의 정의

**유사한 논리:** 어떤 회사 + 딥러닝 = AI 기업? 아니다.

**진정한 AI 기업:** 진정한 AI 기업은 AI가 잘하는 일들을 수행하도록 조직된 회사다.

#### AI 기업의 특징

**1. Strategic Data Acquisition (전략적 데이터 획득)**

**정의:** 데이터를 얻기 위해 전략적으로 움직이는 것

**사례:** 어떤 AI 기업들은 초기에 수익이 나지 않는 제품을 무료로 제공한다. 이유는 데이터를 모으기 위해서다.

**예시:**

- 무료 음성 인식 앱을 제공하여 음성 데이터 수집
- 무료 이미지 라벨링 도구를 제공하여 라벨링된 이미지 데이터 수집
- 나중에 이 데이터로 훨씬 더 가치 있는 서비스를 만든다

**특징:** 전통적인 기업은 즉각적인 수익을 추구하지만, AI 기업은 데이터 수집을 장기 투자로 본다.

**2. Unified Data Warehouse (통합 데이터 웨어하우스)**

**문제 상황 - 데이터 사일로 (Data Silos):** 많은 기업에서 데이터가 각 부서에 흩어져 있다:

- 영업팀이 자기들만의 데이터베이스를 가지고 있다
- 마케팅팀이 또 다른 데이터베이스를 가지고 있다
- 제조팀이 또 다른 시스템을 사용한다

이것을 '사일로(silo)'라고 부른다. 곡물 저장고처럼 각각 분리되어 있다는 의미다.

**AI 기업의 접근:** AI 기업은 모든 데이터를 한 곳에 모은다:

- 데이터 엔지니어가 쉽게 접근할 수 있다
- 서로 다른 데이터 소스 간의 연결점을 발견할 수 있다
- 예: 고객 서비스 데이터와 제품 결함 데이터를 연결하여 품질 문제를 조기에 발견

**이점:** 통합된 데이터가 있으면, 한 부서의 데이터가 다른 부서의 AI 프로젝트에 유용할 수 있다. 사일로에서는 이런 기회를 놓치게 된다.

**3. Pervasive Automation (자동화 기회 포착)**

**마인드셋:** AI 기업은 지속적으로 묻는다: '이 작업을 자동화할 수 있을까?'

**A→B 매핑 찾기:** 모든 업무 프로세스를 보면서 생각한다:

- 입력(A)이 무엇인가?
- 출력(B)이 무엇인가?
- 이것을 머신러닝으로 자동화할 수 있는가?

**예시:**

- 고객 이메일(A) → 적절한 부서로 라우팅(B)
- 이력서(A) → 면접 여부(B)
- 제품 이미지(A) → 카테고리 분류(B)

**문화의 차이:**

- 전통적인 기업: '우리는 항상 이렇게 해왔어'
- AI 기업: '이것을 자동화하면 어떨까?'

**4. New Roles and Division of Work (새로운 역할)**

**Machine Learning Engineer (MLE / 머신러닝 엔지니어)**

**정의:** ML 엔지니어는 머신러닝 모델을 개발하고 배포하는 전문가다.

**전통적 소프트웨어 엔지니어와의 차이:**

**전통적인 소프트웨어 엔지니어:** 명확한 규칙과 로직을 코드로 작성

**ML 엔지니어:**

- 데이터를 수집하고 정제
- 모델을 훈련시키고 평가
- 모델의 성능을 모니터링하고 개선
- 데이터가 바뀌면 모델을 재훈련

**협업 구조:** AI 기업에서는 ML 엔지니어, 데이터 과학자, 전통적인 소프트웨어 엔지니어가 긴밀하게 협력한다.

#### AI Transformation Playbook (AI 변환 플레이북)

[슬라이드 항목 – 간단히 언급만 됨]

기업이 AI 기업으로 변화하기 위한 5단계 로드맵:

1. **Execute pilot projects to gain momentum** (모멘텀 확보를 위한 파일럿 프로젝트 실행)
2. **Build an in-house AI team** (사내 AI 팀 구축)
3. **Provide broad AI training** (광범위한 AI 교육 제공)
4. **Develop an AI strategy** (AI 전략 수립)
5. **Develop internal and external communications** (내부 및 외부 커뮤니케이션 개발)

**언급 내용:** 이 5단계에 대해서는 강의 3주차에 자세히 다룰 예정이다.

### Section 7: What ML can and cannot do

#### 1초의 법칙 (Rule of Thumb)

**핵심 원칙:** 사람이 1초 이내의 생각으로 할 수 있는 일은 대부분 AI로 자동화할 수 있다.

**가능한 것들:**

**1. 다른 차량 위치 파악 (자율주행)** 사람이 운전할 때:

- 전방을 보고
- 0.5초 만에 다른 차가 어디 있는지 파악한다 → AI도 이것을 할 수 있다

**2. 불량품 식별 (제조)** 숙련된 검사원이:

- 제품을 보고
- 1초 이내에 스크래치가 있는지 판단한다 → AI도 훈련시킬 수 있다

**3. 음성 명령 인식** 사람이 '날씨 알려줘'라는 말을 듣고:

- 즉시 의미를 이해한다 → AI도 가능하다

**중요한 단서:** 물론 AI가 이것을 배우려면 많은 양의 훈련 데이터가 필요하다. 하지만 원칙적으로 가능하다.

#### AI가 할 수 없는 것들

**1. 주식 시장 예측**

**질문:** 과거의 주가 데이터만 보고 미래 주가를 예측할 수 있을까?

**답변:** 대부분의 경우 불가능하다.

**이유 1 - Random Walk (랜덤 워크):** 주식 시장은 상당 부분 랜덤 워크 성격을 가진다:

- 내일의 가격은 오늘의 가격과 거의 독립적이다
- 패턴이 있더라도 매우 약하고 불안정하다

**이유 2 - Efficient Market Hypothesis (효율적 시장 가설):** 새로운 정보는 거의 즉시 가격에 반영된다:

- 만약 정말로 미래를 예측할 수 있는 패턴이 있다면
- 많은 사람들이 그것을 이용하여 거래할 것이고
- 그 패턴은 곧 사라질 것이다

**결론:** 과거 가격 차트만 보고는 미래를 예측하기 어렵다. 사람도 못하는 일을 AI도 못한다.

**예외:** 뉴스, 재무제표, 거시경제 지표 등 다른 정보를 함께 사용한다면 조금 더 나을 수 있다. 하지만 여전히 매우 어렵다.

**2. 복잡한 인간 제스처의 의도 파악**

자율주행차가 다른 차를 인식하는 것은 잘 한다. 하지만 다음과 같은 경우는 매우 어렵다:

**어려운 경우들:**

**히치하이커 (Hitchhiker):** 도로변에서 엄지손가락을 치켜드는 사람:

- 차를 태워달라는 의미다
- AI가 이것을 이해하려면 어떻게 해야 할까?

**자전거 수신호:** 자전거 탄 사람이 팔을 옆으로 뻗는다:

- 좌회전 또는 우회전 신호다
- 각도, 방향, 맥락이 중요하다

**경찰의 교통 통제:** 경찰관이 손으로 신호를 보낸다:

- 매우 다양한 제스처가 있다
- 상황에 따라 의미가 다를 수 있다

**왜 어려운가:**

**이유 1 - 데이터 부족:** 이런 상황들은:

- 비교적 드물게 발생한다
- 라벨링된 훈련 데이터를 모으기 어렵다
- 각 제스처의 의미를 정확히 라벨하기 복잡하다

**이유 2 - 복잡한 맥락:** 같은 제스처도 상황에 따라 의미가 다를 수 있다:

- 시간, 장소, 주변 환경
- 다른 시각적 단서들
- 문화적 차이

**결론:** 사람은 이런 복잡한 사회적 신호를 자연스럽게 이해하지만, AI에게는 매우 어렵다. 단순한 1초의 법칙을 넘어서는 문제다.

**3. 교과서 몇 장만 보고 학습하기**

**인간의 학습:** 의대생이 폐렴 진단을 배울 때:

- 교과서 몇 장을 읽는다
- 실제 X-ray 사진 몇 장을 본다
- 그러면 폐렴을 진단할 수 있게 된다

**AI의 학습:** AI가 같은 일을 배우려면:

- 수천, 수만 장의 라벨링된 X-ray가 필요하다
- '폐렴' 또는 '정상'이라고 표시된 데이터

**차이점:**

**인간:**

- 적은 예시로 일반화할 수 있다
- 개념을 추상적으로 이해한다
- 배경 지식을 활용한다

**AI:**

- 많은 예시가 필요하다
- 패턴 매칭에 의존한다
- 맥락 이해가 제한적이다

#### AI의 두 가지 주요 약점

**약점 1 - Learning from small amounts of data** (적은 데이터로 학습하기)

AI는 복잡한 개념을 배우기 위해 많은 데이터가 필요하다. 사람처럼 몇 가지 예시만으로 학습할 수 없다.

**실무적 함의:** AI 프로젝트를 시작하기 전에 자문해야 한다:

- 충분한 훈련 데이터를 확보할 수 있는가?
- 데이터를 수집하는 데 시간과 비용이 얼마나 드는가?

**약점 2 - Generalizing to new types of data** (새로운 유형의 데이터에 일반화하기)

**의료 영상 AI 예시:** 병원 A의 X-ray 기계로 찍은 이미지로 AI를 훈련시켰다고 가정:

- 특정 각도
- 특정 해상도
- 특정 환자 자세

이제 이 AI를 병원 B에서 사용하려고 한다:

- 다른 X-ray 기계
- 약간 다른 촬영 각도
- 이물질(환자가 입고 있는 금속 장신구 등)이 있을 수 있음

→ AI의 성능이 크게 떨어질 수 있다

**인간과의 비교:**

**방사선과 의사:**

- 다른 병원의 X-ray도 잘 읽는다
- 약간의 차이에 적응할 수 있다
- 맥락을 이해하고 조정한다

**AI:**

- 훈련 데이터와 다르면 혼란스러워한다
- 일반화 능력이 제한적이다

**해결 방법:** 이 문제를 완화하려면:

- 다양한 소스의 데이터로 훈련
- 지속적인 모델 업데이트
- 새로운 환경에서 재훈련(fine-tuning)

**AI 프로젝트 계획 시 체크리스트:**

AI 프로젝트를 계획할 때 세 가지 질문:

1. 사람이 1초 이내에 할 수 있는 일인가? → 아마도 가능
2. 충분한 훈련 데이터가 있는가? → 없으면 어려움
3. 배포 환경이 훈련 환경과 비슷한가? → 다르면 조심

이 세 가지 질문에 답하면 AI의 성공 가능성을 대략 예측할 수 있다.

### Section 8: Intuitive explanation of Deep Learning

#### 수요 예측 예시 (Demand Prediction)

**간단한 경우 - 단일 입력:**

티셔츠를 온라인으로 판매한다고 가정한다.

**가장 간단한 모델:**

- Input A: 가격 (Price)
- Output B: 수요 (Demand) - 얼마나 많은 사람이 살 것인가?

**관계:**

- 가격이 낮으면 → 수요가 높다
- 가격이 높으면 → 수요가 낮다

이것을 함수로 표현할 수 있다. 이 함수가 바로 '뉴런(neuron)' 하나다.

**수학적 형태:** 이것은 로지스틱 회귀(logistic regression) 같은 간단한 함수다. 신경망의 가장 기본 단위다.

**복잡한 경우 - 다중 입력:**

이제 더 많은 정보를 고려한다:

- 가격 (Price)
- 배송비 (Shipping cost)
- 마케팅 예산 (Marketing budget)
- 재질 (Material quality)

**딥러닝의 접근:**

이 네 가지 입력이 직접 수요를 예측하는 것이 아니라, 중간 단계를 거친다.

**중간 개념 (Hidden Neurons):**

**뉴런 1 - Affordability (경제성):** '이 제품이 얼마나 경제적인가?'

- 가격과 배송비에 주로 영향을 받는다

**뉴런 2 - Awareness (인지도):** '사람들이 이 제품을 얼마나 알고 있는가?'

- 마케팅 예산에 주로 영향을 받는다

**뉴런 3 - Perceived Quality (인지된 품질):** '사람들이 이 제품을 얼마나 좋다고 생각하는가?'

- 재질과 가격에 영향을 받는다 (비싸면 품질이 좋다고 인식할 수 있음)

이 세 가지 중간 개념이 최종적으로 수요를 예측한다.

**딥러닝의 핵심 - 자동 학습:**

**중요한 특징:** '경제성', '인지도', '품질' 같은 중간 개념을 명시적으로 정의해줄 필요가 없다.

**작동 방식:** 신경망에게 필요한 것:

1. 입력 데이터 (가격, 배송비, 마케팅, 재질)
2. 출력 데이터 (실제 판매량)
3. 충분한 예시들

신경망이 스스로:

- 중간 뉴런들이 무엇을 계산해야 하는지 학습한다
- 입력과 출력 사이의 복잡한 관계를 발견한다

**비유:** 레고 블록을 쌓는 것처럼, 간단한 뉴런들을 여러 층으로 쌓아서 복잡한 함수를 만든다.

#### 얼굴 인식 예시 (Face Recognition)

**컴퓨터가 이미지를 보는 방식:**

컴퓨터는 이미지를 픽셀 밝기 숫자의 거대한 집합으로 본다.

**예시:** 1000x1000 픽셀 이미지 = 1,000,000개의 숫자

**신경망의 단계적 처리:**

**초기 층 (Early Layers):** 모서리(edge)를 감지한다:

- 수직선
- 수평선
- 대각선

**중간 층 (Middle Layers):** 얼굴의 부분(parts)을 감지한다:

- 눈의 형태
- 코의 형태
- 입의 형태

**후반 층 (Later Layers):** 전체 얼굴 형태를 인식한다:

- 모든 부분을 조합
- 특정 사람을 식별

**자동 학습의 원리:**

수많은 이미지 데이터(A)와 정답(B)을 주면:

- 중간의 복잡한 특징 추출 과정을 신경망이 자동으로 학습한다
- 모서리 → 부분 → 전체 얼굴로 이어지는 계층 구조를 스스로 발견한다
- 프로그래머가 "눈을 이렇게 찾아라"고 알려줄 필요가 없다

**딥러닝의 강력함:**

**전통적 접근:** 프로그래머가 명시적으로 규칙을 작성해야 했다:

- "눈은 이런 모양이다"
- "코는 이런 위치에 있다"
- 매우 어렵고 불완전하다

**딥러닝 접근:** 데이터만 충분히 주면:

- 신경망이 스스로 특징을 학습한다
- 인간이 생각하지 못한 패턴도 발견한다
- 더 정확하고 강건하다

## 4. 개념 간 관계

### 계층 구조

```text
AI (Artificial Intelligence)
├── ANI (Artificial Narrow Intelligence) ← 현재 가치 창출의 핵심

│       └── Supervised Learning ← 가장 경제적 가치가 높음
│           ├── Traditional ML
│           └── Deep Learning (Neural Networks)
```

```text
Deep Learning
    ↓ (사용됨)

    ↓                    ↓
```


## 5. 핵심 포인트 요약

### AI의 가치와 범위

- AI는 2033년까지 연간 13조~22조 달러의 가치 창출 예상
- 생성형 AI보다 지도 학습이 더 큰 경제적 가치 창출
- 거의 모든 산업에 영향을 미침
- 현재 가치는 ANI에서 발생, AGI는 먼 미래의 목표

### 지도 학습의 핵심

- 입력(A)에서 출력(B)으로의 매핑 학습
- 가장 경제적 가치가 높은 머신러닝 유형
- ChatGPT 등 LLM도 지도 학습 기반 (다음 단어 예측)

### AI 부상의 이유

- 신경망은 데이터가 많을수록 성능이 계속 향상됨
- 빅데이터의 축적
- GPU 등 컴퓨팅 파워의 발전
- 이 세 가지의 결합이 AI 혁명을 만듦

### 데이터의 중요성

- 데이터는 AI의 핵심 재료
- 정형 데이터와 비정형 데이터로 구분
- 획득 방법: 수동 라벨링, 행동 관찰, 다운로드/파트너십
- 데이터의 양보다 품질이 더 중요
- "Garbage in, garbage out" 원칙

### 용어의 구분

- **Machine Learning**: A→B 소프트웨어 생성
- **Data Science**: 데이터에서 인사이트 추출
- **Deep Learning**: 여러 층의 신경망, 비정형 데이터에 강함
- 신경망은 뇌가 아니라 수학 방정식의 집합

### AI 기업의 조건

- 단순히 AI 기술 사용이 아니라 AI 중심으로 조직화됨
- 전략적 데이터 획득
- 통합 데이터 웨어하우스 (사일로 제거)
- 지속적인 자동화 기회 포착
- 새로운 역할 (ML 엔지니어 등)

### AI의 가능과 불가능

- **1초 법칙**: 사람이 1초 내에 할 수 있는 일은 대부분 자동화 가능
- **불가능한 것들**:
  - 주식 예측 (랜덤 워크, 효율적 시장)
  - 복잡한 제스처 의도 파악 (데이터 부족, 맥락 복잡)
  - 적은 데이터로 학습
- **두 가지 약점**:
  1. 적은 데이터로 복잡한 개념 학습
  2. 새로운 유형의 데이터에 일반화

### 딥러닝의 원리

- 간단한 뉴런을 여러 층으로 쌓아 복잡한 함수 생성
- 중간 개념(Hidden Neurons)을 자동으로 학습
- 충분한 입력-출력 데이터만 있으면 스스로 패턴 발견
- 얼굴 인식 예시: 픽셀 → 모서리 → 부분 → 전체 얼굴
- 프로그래머가 규칙을 명시할 필요 없음
