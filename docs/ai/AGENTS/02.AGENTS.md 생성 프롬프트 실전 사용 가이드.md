# AGENTS.md 생성 프롬프트 실전 사용 가이드

이 문서는 최종 AGENTS.md 생성 프롬프트를 실제 개발 흐름에서 어떻게 사용하는 것이 가장 효과적인지를 정리한 운영 가이드입니다.

## 1. 프로젝트 최초 세팅 (Initial Bootstrap)

프로젝트 시작 시 에이전트에게 프로젝트 전체의 세계관과 헌법을 주입하는 단계입니다.

### ✅ 사용 방법

생성 프롬프트 전체를 AI에게 전달합니다.

바로 아래에 프로젝트의 기술 스택과 핵심 목표를 구체적인 예시와 함께 입력합니다.

### 📝 입력 예시

```text
[AGENTS.md 생성 프롬프트 전체 복사]

### 프로젝트 정보
1. 기술 스택:
   - Frontend: React 18, TypeScript, Tailwind CSS
   - Backend: Node.js, NestJS, PostgreSQL
2. 프로젝트 핵심 목표:
   - 금융 자산 관리 대시보드 구축
   - 우선순위: 보안(Security) > 데이터 정확성 > 성능
3. 주요 디렉토리 구조:
   - /src/features/auth: OAuth2 인증 로직
   - /src/features/wallet: 자산 계산 및 트랜잭션 처리
```

## 2. 새 디렉토리 규칙 추가 (Local AGENTS.md)

새로운 기능(Feature) 폴더를 만들 때, 해당 폴더만의 특수 규칙을 정의합니다.

### ✅ 사용 방법

전체 프롬프트를 다시 붙여넣지 않고, **"기존 원칙"** 을 언급하며 축약 호출합니다.

### 📝 입력 예시

```text
프로젝트 루트(/)의 AGENTS.md 원칙을 기준으로,
/src/features/payments 디렉토리에 대한 Local AGENTS.md를 생성해줘.

특이사항:
- 이 폴더는 외부 결제 SDK(Stripe)를 다룸.
- 결제 실패 시의 예외 처리(Retry 로직)를 반드시 포함할 것.
- Security Rules는 절대 override 불가함.
```

## 3. 일상 개발 세션 (Session Initialization)

매일 작업을 시작할 때 에이전트가 규칙을 잊지 않도록 깨우는 과정입니다. 가장 빈번하게 사용됩니다.

### ✅ 사용 방법

에이전트에게 현재 위치한 디렉토리부터 루트까지의 규칙을 읽고 요약하게 합니다.

### 📝 입력 예시

```text
오늘 작업을 시작하기 전, 루트(/)와 현재 폴더의 AGENTS.md를 읽고
내가 지금 작업할 [결제 연동] 기능에 대해 지켜야 할
MUST / MUST NOT 규칙 3가지만 요약해줘.
```

## 4. 규칙 리팩터링 및 정제 (Refactoring)

규칙이 너무 많아지거나 내용이 추상적일 때 문서를 정비합니다.

### ✅ 사용 방법

자명한 규칙은 지우고, **구체적인 근거(WHY)** 를 보강하도록 지시합니다.

### 📝 입력 예시

```text
현재 AGENTS.md의 규칙 중 "코드를 깔끔하게 작성하라"처럼 당연한 소리는 삭제해줘.

대신 아래 내용을 보강해줘:
- 규칙: API 요청 시 반드시 Timeout을 설정할 것.
- 이유(WHY): 2026-01-15, 특정 API 무한 대기로 서버 가용량 초과 사고 발생.
- 범위: /src/lib/api/**/*
```

## 5. 핵심 운영 원칙 요약

- 한 번 주입, 여러 번 활용: 전체 프롬프트는 최초 1회만 사용하고 이후엔 축약 호출합니다.
- 불필요한 중복 금지: 에이전트에게 이미 학습된 규칙을 반복해서 주입하지 마세요. 토큰 낭비와 집중력 분산을 초래합니다.
- 에이전트 행동 최적화: AGENTS.md는 "있어 보이는 문서"가 아니라 에이전트의 행동을 실제로 바꾸는 규칙이 되어야 합니다.
