# AGENTS.md

## 1. 개요

최근 LLM(대형 언어 모델)과 AI IDE 도구(Cursor, Windsurf 등)의 발전으로 AI는 단순한 코드 자동완성 도구를 넘어 **개발 협업 주체**로 활용되고 있습니다. 이 문서는 다음을 목표로 합니다.

- AI와 협업할 때 **일관된 결과물**을 얻기 위한 기준 정리
- 개인 설정과 프로젝트 지식의 **명확한 분리**
- 특정 도구에 종속되지 않는 **지속 가능한 컨텍스트(Context) 관리 방식** 정립

## 2. AI 코딩의 핵심 개념: Vibe Coding

### 2.1 Vibe Coding이란?

Vibe Coding은 개발자가 모든 세부 로직을 직접 설계하기보다, **설계 의도와 맥락(Context)을 AI에게 전달하고 결과물을 검토·조정하는 협업 방식**을 의미합니다.

- **개발자:** 문제 정의, 의도 전달, 최종 판단 및 리뷰 (Architecture & Review)
- **AI:** 구현, 반복 작업, 코드 초안 생성 (Implementation)

> **Vibe Coding은 설계를 AI에게 위임하는 방식이 아닙니다. 최종 설계 책임과 의사결정은 항상 개발자에게 있습니다.**

### 2.2 왜 Context 관리가 중요한가?

- AI는 프로젝트 전체를 항상 읽을 수 없음 (컨텍스트 윈도우 제한)
- 잘못된 맥락은 **일관성 없는 코드**와 **리뷰 비용 증가**로 직결됨

**핵심:** AI에게 **무엇을, 어디까지 알려줄지**를 구조적으로 관리해야 합니다.

## 3. Cursor Rules (.cursorrules)의 역할

### 3.1 Cursor Rules란?

.cursorrules는 Cursor IDE에서 사용하는 **AI 행동 규칙 파일**입니다.

- AI의 말투, 응답 방식, 코드 스타일 등을 정의
- Cursor의 Chat / Composer 실행 시 **시스템 프롬프트에 가장 가깝게 적용되는 레이어**

### 3.2 무엇에 적합한가? (도구 특화 / 강제 규칙)

Cursor Rules는 **Cursor에 종속된 개인/행동 규칙**에 적합합니다.

- **코드 스타일:** 세미콜론 금지, 네이밍 규칙 강제
- **커뮤니케이션:** "항상 한글로 답변", "설명은 짧게, 코드는 상세하게"
- **동작 원칙:** "코드 수정 후 관련 테스트 코드 실행", "불필요한 파일 수정 지양"

## 4. AGENTS.md의 등장 배경과 목적

### 4.1 AGENTS.md란?

AGENTS.md는 AI 에이전트를 위한 기계 읽기용 README입니다.
AI가 추론으로 보완하지 않고, **그대로 신뢰해야 하는 프로젝트의 기술적 사실(Facts)**을 담는 문서입니다.

### 4.2 왜 필요한가?

1. **도구 파편화 방지:** Cursor, Windsurf, Aider 등 도구가 바뀌어도 동일한 프로젝트 지식 공유 가능
2. **README의 한계 극복:** 사람 중심의 정보(설치, 라이선스)를 배제하고 **AI가 코드를 작성하는 데 필요한 정보**만 집중 제공

## 5. 역할 분담 및 계층적 관리 (Best Practice)

### 5.1 Separation of Concerns (권장 기준)

|          |                                 |                                        |
| -------- | ------------------------------- | -------------------------------------- |
| **구분** | **Cursor Rules (.cursorrules)** | **AGENTS.md**                          |
| **목적** | AI와의 대화/행동 방식 정의      | 프로젝트 지식 및 기술 규약 전달        |
| **성격** | 개인 / 도구 특화 강제 규칙      | 팀 / 프로젝트 공통 가이드라인          |
| **범위** | Cursor 전용                     | 도구 독립 (Standard)                   |
| **예시** | 말투, 답변 언어, 수정 범위 제한 | 아키텍처, 사용 라이브러리, 데이터 스택 |

### 5.2 계층적 Context 탐색 구조

AI 에이전트는 파일을 수정할 때, 일반적으로 다음 순서로 컨텍스트를 참고합니다.

1. **최우선:** 개발자의 현재 채팅 프롬프트
2. **1순위:** 수정 중인 파일과 가장 가까운 위치의 AGENTS.md (지역 규칙)
3. **2순위:** 프로젝트 루트의 AGENTS.md (전역 규칙)
4. **기타:** .cursorrules에 정의된 행동 지침

## 6. 작성 및 운영 가이드 (Tips)

### 6.1 AI가 읽기 좋은 작성법

- **지시형 문장 사용:** "가급적 ~하면 좋다" ❌ → "**~방식을 사용한다**", "**~를 우선 참조한다**" ⭕
- **최소 지식 원칙:** TODO, 과거 논쟁 등 불확실한 내용은 제외하고 **확정된 사실만 기록**

### 6.2 ❌ AGENTS.md에 적합하지 않은 내용

- 임시 결정 사항 및 개인 취향
- 특정 IDE / 특정 도구 전용 명령 (예: /ask 등)
- **방대한 양의 세부 구현 코드** (AI의 컨텍스트를 낭비함)

### 6.3 🛠️ 관리 및 운영 원칙

- **팀 자산화:** AGENTS.md는 프로젝트 코드의 일부로 간주하며, 아키텍처 변경 시 코드 리뷰와 함께 업데이트 여부를 확인합니다.
- **실무 활용 팁:** Cursor의 **Composer(Cmd+I)** 사용 시, @AGENTS.md를 직접 태그하여 AI가 지침을 명확히 인지하게 합니다.
- **개인 최적화 허용:** .cursorrules는 개인별 커스터마이징을 허용하되, 공통 기술 규칙은 AGENTS.md로 일원화합니다.
